**Объяснение паттерна Factory:**

1. **В чём идея паттерна:** Фабрика инкапсулирует логику создания объектов, скрывая детали инициализации от клиентского кода. Клиент работает с абстракцией (интерфейсом), а фабрика решает, какой конкретный объект создать.

2. **Ключевые элементы в коде:**
   - Интерфейс `Transport` определяет общий контракт для всех продуктов
   - Конкретные классы `Truck` и `Ship` реализуют этот интерфейс
   - Класс `TransportFactory` содержит метод `createTransport()`, который решает, какой объект создать

3. **Где виден принцип:** В методе `main()` клиентский код не знает о конкретных классах `Truck` и `Ship` - он работает только с интерфейсом `Transport`. Фабрика решает, какой конкретный объект создать.

4. **Какую проблему решает:** Устраняет жесткую привязку клиентского кода к конкретным классам. Если нужно добавить новый тип транспорта (например, `Airplane`), нужно лишь изменить фабрику, а клиентский код останется неизменным. Это лучше, чем "просто написать напрямую", потому что:
   - Упрощает добавление новых типов продуктов
   - Централизует логику создания объектов
   - Облегчает тестирование (можно подменять фабрику)
