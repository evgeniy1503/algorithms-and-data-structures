**Объяснение паттерна Builder:**

1. **В чём идея паттерна:** Позволяет создавать сложные объекты пошагово, используя один и тот же процесс строительства. Особенно полезен, когда объект имеет много параметров, некоторые из которых необязательны.

2. **Ключевые элементы в коде:**
   - Класс `Computer` (продукт) с приватным конструктором
   - Статический внутренний класс `Builder` с методами для настройки параметров
   - Метод `build()` для финального создания объекта
   - Fluent interface (цепочка вызовов) для настройки параметров

3. **Где виден принцип:** В методе `main()` видно, как можно создавать объекты с разными конфигурациями, используя цепочку вызовов. Каждый метод `Builder` возвращает `this`, что позволяет вызывать методы цепочкой.

4. **Какую проблему решает:** Решает проблему создания объектов с большим количеством параметров, особенно когда многие из них необязательны. Это лучше, чем "просто написать напрямую", потому что:
   - Избегает "телескопических конструкторов" с множеством параметров
   - Делает код более читаемым и понятным
   - Позволяет создавать иммутабельные объекты
   - Упрощает создание объектов с разными конфигурациями
